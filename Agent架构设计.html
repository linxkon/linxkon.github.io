从 Subagent 到 Agent OS：编程 Agent 系统架构演进与工程挑战

随着 Claude Code、Codex CLI、Roo Code 等编程 Agent 的出现，软件开发领域正在从 IDE Assistant 逐步迈向 Autonomous Developer（自动化开发者）。然而，在实践中人们很快发现：单个 LLM Agent 很难稳定完成复杂的软件工程任务。于是系统架构开始出现明显的演进路径：

本文从系统工程角度分析三个核心问题：

  1. 为什么编程 Agent 需要 Subagent / Multi-Agent
  2. 为什么大多数 Agent Framework 最终失败在 orchestration
  3. 为什么未来 AI 系统可能演化为 Agent OS
